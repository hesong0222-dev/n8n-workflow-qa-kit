# WorkflowPatch Browser-Agent Eval Capability Profile

WorkflowPatch is available for small, async browser-agent eval and workflow QA slices using public, synthetic, or redacted scenarios only.

## Fit

- Browser-agent task definition and success/failure rubrics
- Flaky-step notes for navigation, selectors, page state, retries, and timeouts
- Synthetic browser task cases that avoid private sessions and customer data
- Reproducibility notes for demos, eval packs, and handoff reports
- Workflow reliability review for n8n, Make, Zapier, webhooks, Airtable, Google Sheets, Slack, Gmail, APIs, and AI-assisted operations

## Safe Work Model

1. Use a public demo URL, synthetic browser-agent task, redacted screenshots, or a written flow.
2. Agree on a narrow paid slice before work starts.
3. Deliver a concise report with task definition, pass/fail rubric, flaky-step notes, reproduction notes, and synthetic test cases.
4. Avoid customer sessions, proxy credentials, private logs, private account cookies, and production access unless explicitly scoped later.
